# 2026-03-11 首次重构思考

## 发现的问题

1. **src目录职责不清**: `analysis/` 和 `data/` 功能重叠
2. **技术栈虚报**: README列出的HanLP、JioNLP、Plotly等未实际使用
3. **代码质量差**: 大量未实现的功能
4. **脚本混乱**: 命名不规范，依赖关系不清

## 重构思路

### 三层架构设计

```
src/
├── core/              # 纯工具函数，无业务逻辑
│   ├── text_utils.py      # 文本处理
│   └── pinyin_utils.py    # 拼音工具
│
├── features/          # 特征提取（文本 → 向量）
│   ├── rhyme_features.py      # 韵律特征
│   └── sentiment_features.py  # 情感特征
│
├── models/            # 业务模型
│   ├── poetry_classifier.py      # 诗体分类
│   └── social_network_model.py   # 社交网络分析
│
└── visualization/     # 可视化
    └── poetry_visualizer.py
```

### 脚本命名规范化

```
scripts/
├── 01_data_process.py          # 数据处理
├── 02_analysis_*.py            # 分析脚本
├── 03_vis_*.py                 # 可视化脚本
└── 03_generate_all.py          # 一键运行
```

## 关键改进

- **单一职责**: core→features→models，单向依赖
- **消除重复**: 合并 rhyme.py + rhyme_vectorizer.py
- **命名规范**: 01_/02_/03_ 序号化分类
