# 中国古代文学时空图谱：项目重构记录

## 重构背景

基于 `2026-03-11-开发进度.md` 的发现，原项目存在严重的**代码结构混乱**和**文档与代码不一致**问题：

1. **src目录职责不清**: `analysis/` 和 `data/` 功能重叠
2. **技术栈虚报**: README列出的HanLP、JioNLP、Plotly等完全没有使用
3. **代码质量差**: 大量未实现的功能（标记为"shit"）
4. **脚本混乱**: 命名不规范，依赖关系不清

## 重构目标

1. **清晰的目录结构**: core → features → models 三层架构
2. **统一命名规范**: 01_data → 02_analysis → 03_vis 序号化
3. **消除重复代码**: 合并rhyme相关模块，统一文本处理
4. **实际可用**: 所有脚本都能正常运行，生成结果

---

## 重构过程

### Phase 1: 目录结构重构

#### 原结构问题
```
src/
├── analysis/          # 职责不清：既有特征提取又有业务逻辑
│   ├── rhyme.py      # 与 rhyme_vectorizer.py 重复
│   ├── sentiment.py  # 与 vectorization.py 功能重叠
│   └── ...
├── data/             # 与 analysis 功能重叠
│   └── preprocessor.py  # 也在做文本处理
└── visualization/    # 唯一清晰的模块
```

#### 新结构
```
src/
├── core/              # 纯工具函数，无业务逻辑
│   ├── text_utils.py      # 文本处理（繁简转换、清洗）
│   └── pinyin_utils.py    # 拼音工具（平仄、声母韵母）
│
├── features/          # 特征提取（文本 → 向量）
│   ├── rhyme_features.py      # 韵律特征（合并rhyme + rhyme_vectorizer）
│   └── sentiment_features.py  # 情感特征
│
├── models/            # 业务模型（整合特征做分析）
│   ├── poetry_classifier.py      # 诗体/主题分类
│   └── social_network_model.py   # 社交网络分析
│
└── visualization/     # 可视化（保持不变）
    └── poetry_visualizer.py
```

#### 关键改进
- **单一职责**: core只做基础处理，features只做特征提取，models只做业务逻辑
- **消除重复**: 将 rhyme.py + rhyme_vectorizer.py 合并为 rhyme_features.py
- **依赖清晰**: models → features → core，单向依赖

### Phase 2: Scripts命名规范化

#### 原命名问题
```
scripts/

```
scripts/
├── 01_data_process.py          # 数据处理
├── 02_analysis_pipeline.py     # 完整分析流水线
├── 02_analysis_sentiment.py    # 情感分析

├── 02_analysis_meter.py        # 格律分析

#### 命名规则
- `01_`: 数据处理脚本（data）
- `02_`: 分析脚本（analysis），输出JSON到reports/
- `03_`: 可视化脚本（vis），读取02结果生成HTML

### Phase 4: 数据流向规范化

#### 新数据流
```
01_data_process.py
    ↓
data/sample_data/all_poetry.pkl
    ↓
02_analysis_*.py
    ↓
reports/[category]/results.json
    ↓
03_vis_*.py
    ↓
reports/visualizations/*.html
```

#### 各脚本职责

**01_data_process.py**
- 输入: 原始JSON数据（chinese-poetry）
- 处理: 繁简转换、清洗、去重
- 输出: `data/processed_data/all_poetry.pkl`, `data/sample_data/all_poetry.pkl`

**02_analysis_sentiment.py**
- 输入: `data/sample_data/all_poetry.pkl`
- 处理: 情感分析、统计
- 输出: `reports/sentiment_analysis/sentiment_distribution.json`, `author_sentiment.json`

**02_analysis_network.py**
- 输入: `data/sample_data/all_poetry.pkl`
- 处理: TF-IDF相似度、社交网络构建
- 输出: `reports/social_network/network_analysis.json`, `cluster_analysis.json`

**02_analysis_meter.py**
- 输入: `data/sample_data/all_poetry.pkl`
- 处理: 格律识别、平仄分析
- 输出: `reports/meter_analysis/meter_analysis.json`

**03_vis_*.py**
- 输入: 对应的02分析结果
- 处理: 生成Plotly图表
- 输出: `reports/visualizations/*.html`
