# 2026-03-13 数据分层架构设计

## 核心创新：五层数据架构

```
RAW (原始) → BRONZE (清洗) → SILVER (结构化) → GOLD (分析) → OUTPUT (输出)
```

| 层级 | 职责 | 输出格式 |
|------|------|----------|
| RAW | 原始数据，只读 | JSON (submodule) |
| BRONZE | 清洗、合并、繁简转换 | CSV + JSON 元数据 |
| SILVER | 结构化处理、格律提取 | CSV |
| GOLD | 分析结果（相似度、词汇频率） | CSV + JSON |
| OUTPUT | Web/API 输出 | JSON 分片 |

## 关键决策

**已移除的功能**:
- ❌ 社交网络分析（诗词中人名出现概率极低）
- ❌ 情感分析（诗词抽象，准确率低）

**保留的核心分析**:
- ✅ 文本相似度（TF-IDF + Cosine）
- ✅ 词汇频率（jieba分词）
- ✅ 格律分析（模式匹配）

## 纯静态部署约束

由于使用GitHub Pages:
- ❌ 无后端服务
- ❌ 无实时计算
- ❌ 无数据库
- ✅ 所有计算在构建时完成
- ✅ 客户端执行搜索

## 性能优化策略

| 问题 | 解决方案 |
|------|----------|
| 渲染卡顿 | 渐进式加载 + 虚拟滚动 |
| 图表数据过大 | 自动采样（≤5000点） |
| 功能未完成 | FeatureStatus统一管理 |
