# 诗词数据挖掘 - 词向量与N-gram处理管线

## 概述

本项目包含三个核心处理管线，用于对中文诗词数据进行词向量和N-gram分析。

---

## 管线一：词频统计与关键词索引

### 脚本
- `scripts/word_frequency.py`

### 功能
1. 读取 `results/preprocessed/poems_chunk_*.csv`
2. 使用 jieba 对每首诗进行分词
3. 在每个 chunk 文件中添加 `words` 字段
4. 创建关键词到诗 hash 的映射索引

### 输入
- `results/preprocessed/poems_chunk_*.csv`

### 输出
- `results/preprocessed/poems_chunk_*.csv` (更新后的文件，添加words字段)
- `results/keyword_index/keyword_*.json` (关键词索引文件)
- `results/keyword_index/metadata.json` (元数据)

### 运行命令
```bash
python scripts/word_frequency.py
```

### 结果
```
总记录数: 332712
总chunk数: 333
关键词数量: 893638
索引chunk数: 894
输出目录: results\keyword_index
```

## 管线二：N-gram 索引构建

### 脚本
- `scripts/ngram_frequency.py`

### 功能
1. 读取已分词的 chunk 文件
2. 提取 1-gram, 2-gram, 3-gram
3. 构建 N-gram 到诗 hash 的映射索引
4. 支持中间态恢复（checkpoint）

### 输入
- `results/preprocessed/poems_chunk_*.csv` (需要先运行 word_frequency.py)

### 输出
- `results/ngram_index/ngram_1_*.json` (1-gram 索引)
- `results/ngram_index/ngram_2_*.json` (2-gram 索引)
- `results/ngram_index/ngram_3_*.json` (3-gram 索引)
- `results/ngram_index/metadata.json` (元数据)

### 运行命令
```bash
python scripts/ngram_frequency.py
```

### 结果
```
总记录数: 332712
1-gram 数量: 893638
2-gram 数量: 7288080
3-gram 数量: 10201521
```

### N-gram 数量说明
- 1-gram: 词表中不同词的数量
- 2-gram: 不同2词组合数量
- 3-gram: 不同3词组合数量
- 3-gram > 2-gram 是因为组合空间巨大，符合齐夫定律

## 管线三：FastText 词向量训练

### 脚本
- `scripts/fasttext_train.py` - 训练脚本
- `scripts/fasttext_test.py` - 测试脚本

### 功能
1. 使用 jieba 分词结果训练 FastText 模型
2. 支持子词(Subword)处理，解决OOV问题
3. 保存词向量模型和词表

### 输入
- `results/preprocessed/poems_chunk_*.csv`

### 输出
- `results/fasttext/poetry.model` (二进制模型)
- `results/fasttext/vocab.json` (词表)
- `results/fasttext/metadata.json` (元数据)

### 训练参数
```python
VECTOR_SIZE = 100   # 向量维度
MIN_COUNT = 5        # 最小词频 (过滤低频词)
WINDOW = 5           # 上下文窗口大小
EPOCHS = 10          # 训练轮数
```

### 运行命令
```bash
python scripts/fasttext_train.py
python scripts/fasttext_test.py
```

### 结果
```
词表大小: 115,255
向量维度: 100
```

### 优化过程
1. 初始参数: min_count=1, epochs=5
   - 问题: 68%的词只出现1次，导致大量噪声
2. 优化参数: min_count=5, epochs=10
   - 效果大幅提升

### 测试效果

## 词频分析关键发现

### 基本统计
| 指标 | 数值 |
|------|------|
| 总词数 | 11,466,032 |
| 不同词数 | 893,638 |
| 最大词频 | 56,133 |
| 平均词频 | 12.83 |
| 中位数词频 | 1 |

### 词频分布
| 词频区间 | 词数 | 占比 |
|----------|------|------|
| =1 | 610,871 | 68.36% |
| 2-5 | 183,558 | 20.54% |
| 6-10 | 36,671 | 4.10% |
| 11-50 | 41,898 | 4.69% |
| >50 | 20,640 | 2.31% |

### 关键结论
- 68%的词只出现1次（长尾分布）
- 前10%高频词覆盖90%的总词次
- 建议 min_count=5~10 过滤低频噪声

## 技术说明

### N-gram vs Word2Vec vs FastText

| 技术 | 用途 | 优点 | 缺点 |
|------|------|------|------|
| N-gram | 精确匹配检索 | 简单直接 | 不理解语义 |
| Word2Vec | 语义相似 | 捕捉语义 | 不支持OOV |
| FastText | 语义相似+子词 | 支持OOV | 效果略弱于Word2Vec |

### 存储说明
- 词向量矩阵: n × d (893,638 × 100 ≈ 350MB)
- 相似度矩阵: 不需要存储，推理时动态计算
- N-gram 索引: 分chunk存储，每个1000条
