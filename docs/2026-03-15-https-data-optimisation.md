# 2026-03-15 数据结构优化故事

## 问题：6.8GB 的困境

`word_similarity_summary.json` 原始大小 6.8GB，浏览器无法加载。

**根本原因**：每个相似词都有重复的key名
```json
{"word": "圆月", "similarity": 0.92}  // 重复key
```

## 解决：紧凑数组格式

```json
["明月", 5000, [["圆月", 0.92], ["皓月", 0.89], ...]]
```

**结果**：6.8GB → 1.8GB，压缩比 **3.8x**

## 核心洞察

**数据结构 > 传输协议 > 压缩算法**

| 格式 | 大小 | 解析速度 |
|------|------|----------|
| 原始JSON | 6.8GB | 慢 |
| 紧凑JSON | 1.8GB | 快 |
| 二进制 | ~500MB | 极快 |
