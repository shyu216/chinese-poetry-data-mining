# 2026-03-13 Phase 2 静态部署优化

## 关键约束：纯静态部署

| 约束 | 说明 | 解决方案 |
|------|------|----------|
| ❌ 无后端 | 无法运行Flask/Django | 构建时完成所有计算 |
| ❌ 无实时计算 | 无法执行复杂运算 | 预计算所有结果 |
| ❌ 无数据库 | 无法使用MySQL/PostgreSQL | JSON文件存储 |
| ✅ 静态文件 | HTML/CSS/JS直接访问 | 完全支持 |
| ✅ 客户端计算 | 浏览器执行JS | 适合小数据量计算 |

## 静态化设计原则

1. **构建时计算**: 复杂分析在CI/CD阶段完成
2. **分层索引策略**:
   ```
   manifest.json
   ├── author_index.json
   ├── dynasty_index.json
   ├── search_index.json
   └── similarity_index.json
   ```
3. **按需加载**: 首屏只加载必要数据
