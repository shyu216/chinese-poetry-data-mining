# 2026-03-11 AI 协作初探

## 当时的想法

首次尝试与AI协作开发，探索如何让AI助手快速理解项目上下文。

## 关键决策

**简洁优于复杂**
- 不要过度工程化
- 能用简单方法解决的，不要用复杂框架
- 代码要易读易维护

**渐进式改进**
- 先让代码跑起来，再优化
- 小步快跑，频繁验证

## 技术栈选择

Python + Pandas + Plotly + PyTorch/Transformers + NetworkX

数据规模：33万+首诗

## 项目结构初版

```
chinese-poetry-data-mining/
├── data/                      # 数据目录
│   ├── sample_data/          # 采样数据
│   └── processed_data/       # 完整数据
├── src/                       # 源代码
│   ├── core/                 # 核心工具
│   ├── features/             # 特征提取
│   ├── models/               # 分析模型
│   └── visualization/        # 可视化
├── scripts/                   # 执行脚本
├── reports/                   # 分析报告
└── docs/                      # 项目文档
```

## 协作风格确立

- 主要使用中文
- 直接、简洁的沟通
- 重视实际结果
- 给出明确方向，允许AI自主决策实现细节
