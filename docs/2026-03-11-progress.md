# 2026-03-11 项目启动与数据探索

## 当时的尝试

探索 chinese-poetry 开源数据集的结构，了解数据格式差异。

## 数据格式发现

**唐诗结构**
```json
{
  "title": "静夜思",
  "author": "李白",
  "paragraphs": ["床前明月光..."]
}
```

**宋词结构**
```json
{
  "title": "水调歌头",
  "author": "苏轼",
  "paragraphs": [...],
  "rhythmic": "水调歌头"
}
```

**古文结构**（chinese-gushiwen）
```json
{
  "title": "将进酒",
  "dynasty": "唐代",
  "writer": "李白",
  "content": "...",
  "remark": "注释",
  "translation": "白话翻译",
  "shangxi": "赏析"
}
```

## 开发优先级初定

```
Phase 1: 数据探索 (1-2天)
├── 加载数据、了解结构
├── 基础统计 (作者分布、朝代分布)
└── 数据质量检查

Phase 2: 预处理 (2-3天)
├── 繁简转换
├── 拼音标注
└── 数据清洗、去重
```

## 关键发现

不同数据源的字段命名不一致（author vs writer），需要统一Schema。
