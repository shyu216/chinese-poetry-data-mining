# 中国诗词数据挖掘项目

## 📋 项目概述

一个基于纯静态架构的中国诗词数据挖掘与可视化项目。通过构建时计算完成所有数据分析，生成静态索引，客户端直接执行搜索和可视化。

### 核心设计理念

**完全去后端化**：
- ❌ 无需 Flask/Django 服务器
- ❌ 无需 MySQL/PostgreSQL 数据库
- ❌ 无需实时计算
- ✅ 所有分析在构建时完成
- ✅ 客户端执行所有搜索
- ✅ 预计算的静态索引

**优势**：零成本部署（GitHub Pages）、极快响应速度、无限扩展性

## 🏗️ 架构概览

```
构建时计算                    运行时
─────────────────────────  ────────────────────
数据清洗 → 结构化 → 分析 → 索引构建 → 前端展示
     ↓         ↓        ↓        ↓
  Bronze   Silver   Gold    Output    GitHub Pages
```

### 数据分层

| 层级 | 说明 | 格式 | 输出文件 |
|------|------|------|----------|
| Bronze | 清洗合并后的数据 | CSV + JSON | v1_poems_merged.csv |
| Silver | 结构化数据，包含格律提取 | CSV 分片 | v2_poems_*.csv |
| Gold | 分析结果（相似度、词频） | JSON | v3_*.json |
| Output | Web 就绪的静态文件 | JSON 分片 + HTML | data/output/web/ |

### 数据规模

- 诗词数量：约 20 万首（全唐诗 + 宋词）
- 作者数量：数千位诗人
- 词汇表大小：数万个词汇
- 分块数量：约 200 个 chunks（每块 1000 首）

## 🔄 数据处理 Pipeline

### 步骤 1: 数据清洗 (01_clean.py)

- 加载 chinese-poetry submodule 的原始数据（全唐诗、宋词）
- 统一诗/词/曲字段名
- 繁简转换
- 基于内容 hash 去重
- 生成采样数据

**关键技术**：JSON 批量加载、MD5 hash 去重、Pandas DataFrame 处理

**输出**：`data/bronze/v1_poems_merged.csv`

### 步骤 2: 结构化处理 (02_structure.py)

- 解析诗句结构（按行分割）
- 提取格律模式（如 "7,7,7,7" 表示七言绝句）
- 判断是否为格律诗（绝句、律诗）
- 分类诗体类型（五言绝句、七言律诗等）
- 使用 ChunkManager 分块处理大规模数据

**关键技术**：正则表达式清理、格律模式识别、诗体分类算法

**输出**：`data/silver/chunks/v2_poems_*.csv`（分片存储）

### 步骤 3: 词汇频率分析 (03_analyze_words.py)

- 使用 jieba 分词统计每位作者的高频词汇
- 词性标注（名词、动词、形容词）
- 排除停用词（虚词、助词等）
- 生成全局词频统计
- 支持断点续传（中间状态缓存）

**关键技术**：jieba 分词 + 词性标注、Counter 词频统计、停用词过滤

**输出**：`data/gold/v3_word_frequency.json`

### 步骤 4: 文本相似度分析 (04_analyze_similarity.py)

- 为每位作者构建文档（所有诗词拼接）
- 使用 TF-IDF 向量化
- 计算 Cosine Similarity（余弦相似度）
- 构建相似度矩阵
- 提取每位作者的 Top N 相似作者
- 构建网络边（用于 D3.js 可视化）

**关键技术**：scikit-learn TfidfVectorizer、Cosine Similarity 计算

**输出**：`data/gold/v3_text_similarity.json`

### 核心模块

#### ChunkManager 分块管理器 (chunk.py)

- `split_to_chunks()`: 将 DataFrame 分割成多个 chunk 文件
- `read_chunk()`: 读取指定 chunk
- `read_all_chunks()`: 读取所有 chunk 并合并
- `iter_chunks()`: 迭代所有 chunk（内存友好）
- `get_chunk_count()`: 获取 chunk 总数

#### WordFrequencyAnalyzer 词汇频率分析器 (word_frequency.py)

- jieba 分词 + 词性标注
- 统计每位作者的 Top N 高频词
- 过滤停用词和短词
- 生成词性分布
- 支持作者用词习惯比较

#### TextSimilarityAnalyzer 文本相似度分析器 (text_similarity.py)

- TF-IDF 向量化作者文档
- Cosine Similarity 计算相似度
- 构建相似度矩阵
- 提取相似作者列表
- 生成网络边数据

## 📦 索引构建

## 🎨 前端展示

### 技术栈

- **原生 JavaScript** - 不使用 React/Vue 等框架
- **D3.js v7** - 数据可视化库（力导向图）
- **原生 HTML5 + CSS3** - 现代化 UI 设计
- **GitHub Pages** - 纯静态托管

### 性能优化

- **倒排索引** - 客户端搜索引擎
- **Web Workers** - 处理大索引文件
- **渐进式加载** - 分片加载诗词数据
- **虚拟滚动** - 最多渲染 100 个 DOM 节点

### 功能页面

| 页面 | 功能 | 文件 |
|------|------|------|
| 相似度网络图 | D3.js 力导向图，诗人关系网络，可调节阈值和朝代筛选 | similarity-network.html |
| 作者相似度搜索 | 基于预计算的相似度矩阵，显示相似度分数和共同词汇 | author-similarity.html |
| 智能搜索 | 全文搜索、多关键词组合、按作者/朝代筛选、搜索缓存 | search.html |
| 格律大全 | 按格律类型浏览、句数筛选、统计信息展示 | meter-gallery.html |

### 索引文件

- `search_index.json` - 倒排索引
- `author_index.json` - 作者索引
- `author_similarity_index.json` - 作者相似度矩阵
- `author_network.json` - 网络图数据
- `dynasty_index.json` - 朝代索引
- `recommendation_index.json` - 推荐索引
- `pos_index.json` - 词性标注索引

## 🚀 部署流程

```bash
# 1. 数据流水线
python scripts/steps/01_clean.py
python scripts/steps/02_structure.py
python scripts/steps/03_analyze_words.py
python scripts/steps/04_analyze_similarity.py

# 2. 构建静态索引
python scripts/index/build_all_indexes.py

# 3. 导出 Web 文件
python scripts/export/web.py

# 4. 本地测试
python -m http.server 8080 --directory data/output/web
```

## ✨ 技术亮点

1. **纯静态部署** - 无需后端服务，GitHub Pages 直接部署
2. **构建时计算** - 所有分析在本地完成，生成静态 JSON
3. **客户端搜索** - 使用倒排索引 + Web Workers 的纯 JavaScript 搜索
4. **分块处理** - ChunkManager 处理大规模数据，降低内存占用
5. **断点续传** - 词汇分析支持中间状态缓存
6. **Bronze/Silver/Gold 分层** - 清晰的数据血缘和版本管理
