# 2026-03-15 IndexedDB缓存方案

## 问题

每次刷新页面都重新加载33万首诗词数据，体验差。

## 方案选择

| 方案 | 容量 | 适用场景 |
|------|------|----------|
| localStorage | ~5-10MB | 小数据 |
| IndexedDB | 几百MB-几GB | 大数据、离线应用 |

**选择**: IndexedDB - 容量大、异步不阻塞UI

## 存储结构

```
poem-cache (DB)
├── chunks - 分块摘要数据
├── chunkDetails - 诗词详情
├── index - 索引数据
└── metadata - 会话元数据
```

## 数据流

**首次访问**:
加载 → 缓存到IndexedDB → 记录loadedChunkIds

**刷新页面**:
从IndexedDB恢复 → 无需网络请求 → 立即显示

**加载新chunk**:
先检查缓存 → 有则直接返回 → 无则网络请求

## 核心优势

- 刷新后秒开
- 减少网络请求
- 支持离线浏览已缓存内容
