# Web 开发记录 - 浏览器本地存储方案

**日期**: 2026-03-15  
**主题**: IndexedDB 缓存架构设计与实现

---

## 背景

诗词浏览页面需要处理 332,712 首诗词的大数据量，每次刷新页面都重新加载数据体验不佳。需要实现浏览器本地持久化存储。

---

## 浏览器存储方案对比

| 方案 | 容量 | 特点 | 适用场景 |
|-----|------|------|---------|
| **localStorage** | ~5-10MB | 同步、永久存储、字符串 only | 小数据、配置 |
| **sessionStorage** | ~5-10MB | 同步、标签页关闭清除 | 临时数据 |
| **IndexedDB** | 几百MB-几GB | 异步、结构化数据、支持索引 | 大数据、离线应用 |
| **Cache API** | 取决于磁盘 | 配合 Service Worker | 缓存请求响应 |

**选择**: IndexedDB - 容量大、支持结构化数据、异步不阻塞 UI

## 架构设计

### 存储结构

```
IndexedDB: poem-cache
├── chunks (ObjectStore)          # 分块摘要数据
│   └── { id, poems[], timestamp }
├── chunkDetails (ObjectStore)    # 诗词详情数据
│   └── { id, poems: Map, timestamp }
├── index (ObjectStore)           # 索引数据
│   └── { key: 'main', data, timestamp }
└── metadata (ObjectStore)        # 会话元数据
    └── { key: 'session', loadedChunkIds[], timestamp }
```

### 数据流

```
首次访问:
  加载 index.json → 缓存到 IndexedDB
  加载 chunk 0 → 缓存到 IndexedDB
  记录 loadedChunkIds → 缓存到 IndexedDB

刷新页面:
  从 IndexedDB 读取 index
  从 IndexedDB 读取 loadedChunkIds
  从 IndexedDB 恢复已加载的 chunks
  无需网络请求，立即显示

加载新 chunk:
  先检查 IndexedDB 缓存
  如果有，直接返回
  如果没有，网络请求 → 缓存到 IndexedDB
```

## 核心代码实现

### 1. IndexedDB 封装 (usePoemCache.ts)

```typescript
import { openDB, type DBSchema, type IDBPDatabase } from 'idb'

interface PoemCacheSchema extends DBSchema {
  chunks: {
    key: number
    value: { id: number; poems: PoemSummary[]; timestamp: number }
  }
  chunkDetails: {
    key: number
    value: { id: number; poems: [string, PoemDetail][]; timestamp: number }
  }
  index: {
    key: string
    value: { data: any; timestamp: number }
  }
  metadata: {
    key: string
    value: { loadedChunkIds: number[]; timestamp: number }
  }
}

const DB_NAME = 'poem-cache'
const DB_VERSION = 1

// 缓存 chunk 摘要
export async function cacheChunkSummaries(chunkId: number, poems: PoemSummary[]) {
  const db = await getDB()
  await db.put('chunks', { id: chunkId, poems, timestamp: Date.now() })
}

// 获取缓存的 chunk 摘要
export async function getCachedChunkSummaries(chunkId: number): Promise<PoemSummary[] | null> {
  const db = await getDB()
  const cached = await db.get('chunks', chunkId)
  return cached?.poems || null
}

// 缓存 metadata（已加载的 chunk IDs）
export async function cacheMetadata(loadedChunkIds: number[]) {
  const db = await getDB()
  await db.put('metadata', { key: 'session', loadedChunkIds, timestamp: Date.now() })
}

// 获取缓存的 metadata
export async function getCachedMetadata(): Promise<number[] | null> {
  const db = await getDB()
  const cached = await db.get('metadata', 'session')
  return cached?.loadedChunkIds || null
}
```

### 2. 集成到 usePoems.ts

```typescript
import {
  cacheChunkSummaries, cacheIndex, cacheMetadata,
  getCachedChunkSummaries, getCachedIndex, getCachedMetadata
} from './usePoemCache'

// 加载 chunk 时自动缓存
const loadChunkSummaries = async (chunkNum: number): Promise<PoemSummary[]> => {
  // 1. 检查内存缓存
  if (chunkCache.value.has(chunkNum)) {
    return chunkCache.value.get(chunkNum)!
  }
  
  // 2. 检查 IndexedDB 缓存
  const cached = await getCachedChunkSummaries(chunkNum)
  if (cached) {
    chunkCache.value.set(chunkNum, cached)
    return cached
  }
  
  // 3. 网络请求
  const response = await fetch(`/data/preprocessed/poems_chunk_${chunkId}.csv`)
  const poems = parseCSV(await response.text())
  
  // 4. 缓存到内存和 IndexedDB
  chunkCache.value.set(chunkNum, poems)
  await cacheChunkSummaries(chunkNum, poems)
  
  // 5. 更新 metadata
  if (!loadedChunkIds.includes(chunkNum)) {
    loadedChunkIds.push(chunkNum)
    await cacheMetadata(loadedChunkIds)
  }
  
  return poems
}

// 页面加载时恢复缓存
const restoreCachedChunks = async (): Promise<number[]> => {
  const cachedIds = await getCachedMetadata()
  if (cachedIds && cachedIds.length > 0) {
    loadedChunkIds = [...cachedIds]
    // 预加载 chunks 到内存
    for (const chunkId of cachedIds) {
      const cached = await getCachedChunkSummaries(chunkId)
      if (cached) {
        chunkCache.value.set(chunkId, cached)
      }
    }
  }
  return loadedChunkIds
}
```

### 3. 页面初始化 (PoetsView.vue)

```typescript
onMounted(async () => {
  // 1. 先恢复 IndexedDB 缓存
  const cachedChunkIds = await restoreCachedChunks()
  loadedChunkIds.value = cachedChunkIds
  
  // 2. 获取基础信息
  const stats = await getStats()
  
  // 3. 如果有缓存，直接使用；否则初始加载
  if (loadedChunkIds.value.length === 0) {
    await initialLoad() // 加载第一个 chunk
  } else {
    await loadPoems() // 使用缓存的数据
  }
})
```

## 性能优化

### 1. 双层缓存策略

```
内存缓存 (Map) → 快速访问
     ↓
IndexedDB → 持久化存储
     ↓
网络请求 → 最终回退
```

### 2. 延迟加载

- 只加载用户需要的 chunks
- 详情数据按需加载（不缓存到内存）

### 3. 批量操作

```typescript
// 下载全部 chunks
for (let i = 0; i < totalChunks; i++) {
  await loadChunkSummaries(i)
  progress.value = Math.round(((i + 1) / totalChunks) * 100)
}
```

## 存储容量估算

| 数据类型 | 单条大小 | 数量 | 总大小 |
|---------|---------|------|--------|
| index.json | 60KB | 1 | 60KB |
| chunk 摘要 | ~80KB | 333 | ~26MB |
| chunk 详情 | ~200KB | 333 | ~66MB |
| **总计** | - | - | **~100MB** |

IndexedDB 通常支持 50MB-几百MB，足够存储全部数据。
